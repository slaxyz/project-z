HOW WE CAN IMPLEMENT ANIMATIONS
Recommended production method:
1. Use small dedicated scripts per component
2. Drive animation by scale / position / alpha
3. Prefer a tween tool such as DOTween for speed and clean iteration
4. Keep animation values exposed in the Inspector so they can be tuned visually

Do not:
- hard-code huge animation systems too early
- mix gameplay logic and animation logic in one script


ANIMATION EXAMPLES TO KEEP IN MIND
Marvel Rivals style vibrating duplicate button
- duplicate the visual 1-2 times behind the button
- offset them slightly
- animate tiny position jitter + alpha flicker
- keep gameplay hitbox only on the main button

Button bounce on click
- quick scale down
- quick scale up slightly above normal
- settle back to 1
- very short, around 0.12 to 0.18 sec total feel

Collection hero card default <-> selected
- selected frame fades/scales in
- card scales from default size to selected size
- dark overlay fades out when selected
- keep this very readable and snappy, not floaty

WIP SCENE WORKFLOW
For every new UI scene:
1. Duplicate the base scene
2. Rename it with WIP suffix
3. Build the screen manually with Unity UI
4. Do not develop the UI in code unless explicitly requested

Always add these Empty GameObjects at scene root when working in a WIP scene:
- DISABLE_DEBUG_UI
- DISABLE_LOBBY_UI

Purpose:
- DISABLE_DEBUG_UI hides the debug overlay
- DISABLE_LOBBY_UI hides the lobby overlay that can appear on unrecognized WIP scenes

Default workflow with Clement:
- duplicate scene with WIP suffix
- add Disable markers
- create hierarchy manually
- build UI manually in Unity
- validate visually
- only then connect logic if needed
